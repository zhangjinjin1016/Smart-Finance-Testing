# 智慧理财 - 功能测试项目（手工 + 三角色UI自动化）

## 项目背景

智慧理财是一个P2P借贷平台，有借款人、投资人、平台审核员三个角色。核心业务包括额度申请、借款发布、投资、充值提现等。
本项目包含**手工功能测试**和**三角色 UI 自动化冒烟测试**，完整覆盖核心业务闭环（借款申请 → 后台审核 → 投资）。

## 测试范围

- 注册、登录、开户
- 额度申请（额度类型：信用/担保/抵押/流转，金额1~1,000,000）
- 额度审核（通过金额 ≤ 申请金额）
- 借款申请（信用标：金额100~200,000且为50倍数，年利率3%~10%）
- 借款业务流程（借款人 → 审核员 → 发布标 → 投资列表）
- 投资（投资金额10倍数，余额不足跳转充值）
  
## 运行脚本
借款人冒烟测试，审核员冒烟测试，投资人冒烟测试。

## 脚本功能
借款人：自动登录 → 填写借款申请 → 提交 → 断言成功

审核员：登录后台 → 进入待审核列表 → 断言存在待审记录

投资人：登录 → 进入投资列表 → 选择标的 → 输入金额 → 投资 → 断言成功

## 技术亮点
使用 WebDriverWait 显式等待，避免固定 sleep 导致的不稳定性
每个角色独立脚本，可单独执行并生成 HTML 报告
集成 pytest-html，报告包含用例状态、耗时、断言详情

## 测试成果

- 设计测试用例86条（正向、逆向、边界值），计划执行70条（P0/P1/P2优先级），实际执行70条，通过65条，通过率92.9%
- 发现5个缺陷（严重1个、一般3个、轻微1个），已修复4个，修复率80%

**典型缺陷**：
- 借款投资密码输入5位纯数字，预期提示6~16位字母数字组合，实际发布借款成功
- 筹标期限设为256天（需求最大值），无错误提示且申请失败
- 项目名称输入25个汉字提示超长，但需求要求50字符以内
