# 智慧理财 - 功能测试项目（手工 + 三角色UI自动化）

## 项目背景

智慧理财是一个P2P借贷平台，有借款人、投资人、平台审核员三个角色。核心业务包括额度申请、借款发布、投资、充值提现等。
本项目包含**手工功能测试**和**三角色 UI 自动化冒烟测试**，完整覆盖核心业务闭环（借款申请 → 后台审核 → 投资）。

## 测试范围
- 注册、登录、开户
- 额度申请（额度类型：信用/担保/抵押/流转，金额1~1,000,000）
- 额度审核（通过金额 ≤ 申请金额）
- 借款申请（信用标：金额100~200,000且为50倍数，年利率3%~10%）
- 借款业务流程（借款人 → 审核员 → 发布标 → 投资列表）
- 投资（投资金额10倍数，余额不足跳转充值）
  
## 运行脚本
借款人冒烟测试，审核员冒烟测试，投资人冒烟测试。

## 脚本功能
每个角色脚本可单独执行，覆盖完整业务链路：
- 借款人脚本：登录前台 → 填写额度 / 借款申请并提交 → 断言提交成功
- 审核员脚本：登录管理后台 → 进入待审核列表 → 断言存在新建借款单据
- 投资人脚本：登录投资页面 → 选择审核通过标的 → 输入金额完成投资 → 断言投资成功
- 执行后自动生成对应角色独立 HTML 测试报告。

## 技术亮点
- 脚本稳定性：全程使用 WebDriverWait 显式等待元素加载，摒弃固定 sleep，规避页面加载波动导致执行失败；
- 模块化分层：借款人 / 审核员 / 投资人三套脚本完全解耦，可按需单独执行，适配分模块快速冒烟；
- 可视化报告：集成 pytest-html 插件，自动生成分角色报告，包含用例状态、执行耗时、断言失败详情；
- 金融场景适配：针对金额、年限、账号密码、字符长度设计多层边界校验，匹配金融系统强数据约束需求。

## 测试成果
- 手工功能测试：设计 86 条覆盖全模块用例，执行 70 条核心 P0/P1 高优用例，整体通过率 92.9%；累计发现 5 处缺陷（1 严重 / 3 一般 / 1 轻微），推动开发修复 4 处，缺陷修复率 80%；
- 自动化落地价值：迭代版本冒烟无需人工重复操作全链路，三角色端到端流程 2 分钟完成校验，释放人力专注异常场景、边界手工测试；
- 风险拦截：提前发现密码校验、数值阈值、字符长度 3 类金融高危校验缺陷，规避线上资金、账号安全风险。

## 典型缺陷：
- 密码校验缺陷：需求规定 6~16 位字母数字组合，仅输入 5 位纯数字无拦截，可正常发布借款标的，存在用户账号被盗安全风险（严重缺陷）；
- 筹标年限阈值缺失：需求上限 256 天，输入最大值无报错拦截，单据正常提交，易引发业务计息逻辑异常；
- 项目名称字符校验错误：需求上限 50 字符，仅 25 个汉字就提示超长，校验规则配置错误影响用户操作。
